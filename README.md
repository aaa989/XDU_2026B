# XDU_2026B
西安电子科技大学 2026 年 B 类课程实验：基于 Node.js 的雾霾数据可视化检测系统
📋 项目简介
本项目是一个轻量级的雾霾数据检测与可视化平台，通过调用官方空气质量 API 获取实时数据，在前端进行图表绘制和数据展示。实验整体难度适中，核心在于理解前后端数据交互流程和API 调用规范。

🎯 实验重点与核心知识点
本实验的考察重点不在于复杂的算法实现，而在于掌握 Web 开发的基础数据流转逻辑：
1. API 调用与传参机制
  数据来源：从官方空气质量 API 接口获取实时雾霾数据
  传参方式：理解 GET/POST 请求的参数传递方法
  参数流向：明确参数从哪里发起、传递到哪里、如何被后端接收
  认证方式：掌握 API 密钥 (API Key) 的正确使用和安全管理
2. 后端数据处理
  接收前端请求并转发至官方 API
  对 API 返回的原始数据进行解析和清洗
  处理异常情况（如 API 调用失败、数据格式错误）
  将处理后的数据以标准 JSON 格式返回给前端
3. 前后端数据交互
  后端端口的配置与监听
  前端如何发起异步请求获取后端数据
  跨域问题的产生原因与解决方案
  响应数据的接收与解析
4. 前端数据可视化
  将后端返回的 JSON 数据转换为图表可用格式
  使用图表库绘制折线图、柱状图等可视化图表
  实现数据的实时更新和动态展示
  页面布局与用户交互设计

🚀 快速启动指南
环境要求
Node.js 14.0 及以上版本
启动步骤
在项目文件夹顶部的地址栏中输入 cmd 并回车，或在文件夹空白处右键选择 "在此处打开命令窗口"
在终端中输入以下命令启动服务：
node server.js
终端会自动弹出前端访问地址，复制该地址到浏览器中打开即可
浏览器兼容性
✅ 推荐使用：Microsoft Edge
❌ 不推荐：联想系统自带浏览器（可能存在兼容性问题）
其他主流浏览器（Chrome、Firefox）均可正常使用

❓ 常见问题
Q1: 运行 node server.js 提示 "command not found" 怎么办？
A: 请先安装 Node.js 环境，下载地址：https://nodejs.org/
Q2: 启动服务后浏览器无法访问？
A:检查终端显示的端口号是否被其他程序占用
尝试更换浏览器（推荐 Edge）
关闭电脑防火墙或允许 Node.js 通过防火墙
Q3: API 调用失败，页面显示 "数据加载失败"？
A:检查你的 API 密钥是否正确配置
确认网络连接正常
查看后端终端的错误输出信息
